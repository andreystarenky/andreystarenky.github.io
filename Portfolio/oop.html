Inheritance
 Inheritence is the concept that involves objects taking on properties of already existing objects or classes.
This uses the parent-child class relationship, also known as Superclass - Subclass.
In Java, inheriting a class is used with the keyword extends (ex. Public class Cat extends Animal).
A Subclass can access any of the super's public variables and methods. To protect variables from being accidentally modified, the Superclass's vairables are set to private,
and getter/setter methods are used to obtain and change the value of the variables.
A Subclass can also Override any of the super's methods if needed.

Encapsulation
 Encapsulation is the structured design of a class. It incorporates a set attributes along with a set of methods to be able to retrieve and change these variables
As mentioned previously, these methods are called "getters" and "setters" than can return or change the values
Encapsulation is a basic concept that is key for all Object Oriented Programming

Abstraction
 Abstraction is a key concept of Object-Oriented Programming that is incorporated into the blueprint factor of Superclasses. Abstraction is used to tell the subclasses which methods they
need to incorporate individually. This ensures that the child classes are following the template provided by the Abstract class and can have all necessary behaviours.
Methods apply abstraction by using the key word abstract (ex. public abstract void move();).
Abstract methods may not have any code in them at the Parent level. Any class that incorporates abstract methods needs to include the key word abstract before class (ex. public abstract class Animal())
Interfaces are classes that only have abstract methods, and are signified by the key word Interface before class (ex. public interface LifeCycle())
